# 10_Meta

<div class="sb-banner sb-banner--green"><strong>Governance layer.</strong> Shared rules begin in agent-core and synchronize outward to each adapter.</div>

**Purpose:** The vault’s schema, policies, governance, compatibility, and validation documentation.

**What belongs:** Canonical agent rules, taxonomy, safety, source policy, workflow index, and tooling research.

**What does not belong:** Daily work, project content, raw sources, or generated reports.

**Writers:** Both, but shared-rule changes require adapter synchronization and human review.

**Examples:** `agent-core.md`, `safety-policy.md`.
