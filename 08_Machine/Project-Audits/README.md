# Project-Audits

**Purpose:** Read-only audits of project memory or ingested codebases.

**What belongs:** Scope, evidence, findings, stale/dead/mock code candidates, risks, and next steps.

**What does not belong:** Changes to the original project or unsupported accusations.

**Writers:** AI writes; humans review.

**Examples:** `2026-07-13 - project-x audit.md`.
