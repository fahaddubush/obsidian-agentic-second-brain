# 11_Templates

**Purpose:** Versioned Markdown and JSON templates for consistent note creation and machine-verifiable manifests.

**What belongs:** YAML schemas and reusable note skeletons.

**What does not belong:** Completed notes, live project state, or tool-specific binary templates.

**Writers:** Both; shared template changes should remain backward-compatible or documented.

**Examples:** `llm-session-summary.md`, `context-pack.md`, `ingestion-manifest.json`.
