# 06_Procedural-Memory

**Purpose:** Reusable instructions for doing work reliably.

**What belongs:** Checklists, commands, setup guides, runbooks, and troubleshooting procedures.

**What does not belong:** One-time session logs or untested commands presented as safe.

**Writers:** Both; commands that can change state must state prerequisites and risks.

**Examples:** `deploy-staging.md`, `recover-git-branch.md`.
